comment/get

Retrieve a single Instagram comment by its ID. Returns the comment text, author username, timestamp, like count, and visibility (hidden/unhidden) status.

When to Use

  • Comment moderation pipeline: After a webhook fires for a new comment event, retrieve full comment details to evaluate its content — text, author, timestamp — before deciding to hide, respond, or escalate to a human moderator.
  • Sentiment analysis: Fetch specific comments identified by ID from a webhook payload and pass the text field into an AI sentiment analysis node.
  • Customer service routing: Retrieve comment details to check whether it contains a service request keyword, then route to the appropriate CRM case creation workflow.
  • Audit logging: Before hiding or deleting a comment, retrieve and log the full comment data to an audit database for compliance records.
  • Reply context: Fetch the original comment text before crafting an automated or human reply, ensuring the response is contextually accurate.

Configuration

Connection

FieldRequiredDescription
accessTokenRequiredLong-lived Instagram access token stored in BizFirst Credentials Manager.
igUserIdRequiredYour Instagram Business/Creator account ID.

Operation

FieldRequiredDescription
resourceRequiredMust be comments.
operationRequiredMust be get (note: the underlying API uses a direct node lookup by comment ID).
commentIdRequiredThe Instagram comment object ID to retrieve.
fieldsOptionalComma-separated fields: id,text,username,timestamp,like_count,replies_count,hidden.

Sample Configuration JSON

{
  "resource": "comments",
  "operation": "get",
  "accessToken": "{{ $credential.instagram.accessToken }}",
  "igUserId": "17841400000000000",
  "commentId": "{{ $trigger.body.value.commentId }}",
  "fields": "id,text,username,timestamp,like_count,hidden"
}

Node Policies & GuardRails

  • Rate limit: 200 calls per user per hour. In high-volume comment monitoring workflows, batch comment fetches where possible or use webhook-driven architecture to avoid polling.
  • GDPR: Comment text and usernames are personal data. Apply data minimisation — only store what is necessary for the business purpose. Honour deletion requests by purging stored comment data.
    • accessToken is requiredaccessToken field is missing.
  • Deleted comments: If the comment was deleted before retrieval, the operation routes to the error port with Graph API error 100. Handle the error port gracefully in moderation workflows.
