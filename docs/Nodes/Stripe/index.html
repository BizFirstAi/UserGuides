Stripe

Create payment intents, manage customers, handle subscriptions, and process refunds — directly within BizFirst workflows.

What this node does: The Stripe node integrates BizFirst workflows with the Stripe Payments API. Automate billing, subscription lifecycle, customer management, and financial reconciliation — connecting payment events to your business processes without writing Stripe SDK code.

Supported Operations

OperationDescription
create_payment_intentCreate a PaymentIntent for a specified amount and currency. Returns a client_secret for client-side confirmation.
create_customerCreate a Stripe Customer record with email, name, and metadata. Returns the Stripe customer ID for future billing operations.
retrieve_customerRetrieve a Stripe Customer object by ID including attached payment methods and subscription status.
create_subscriptionSubscribe a customer to a plan/price. Handles trial periods, billing cycles, and metadata.
cancel_subscriptionCancel an active subscription immediately or at period end.
create_refundIssue a full or partial refund for a completed charge or PaymentIntent.
retrieve_payment_intentFetch the current status of a PaymentIntent to verify payment completion.
create_invoiceCreate and optionally finalize and send a Stripe invoice to a customer.

Key Capabilities

  • Full payment lifecycle automation — create, confirm, capture, refund
  • Customer and subscription management without manual Stripe dashboard work
  • Supports all major currencies via Stripe's multi-currency infrastructure
  • Metadata support on all objects for business data correlation
  • Test mode / live mode toggle via credential configuration
  • Idempotency key support to prevent duplicate charges on retry
  • Invoice creation with line items, tax rates, and automatic finalization

Business Benefits

Automated Billing Workflows

Connect order completion to payment collection without manual steps. When a service is provisioned, automatically create a PaymentIntent, notify the customer, and capture funds — all within a single BizFirst workflow.

Subscription Lifecycle Management

Handle upgrades, downgrades, and cancellations programmatically. When a customer requests a plan change through your application, a BizFirst workflow updates the Stripe subscription and notifies downstream systems (CRM, provisioning, email) in one atomic flow.

Refund Automation

Trigger refunds from approval workflows. A customer service agent approves a refund request in BizFirst's HIL interface, and the Stripe refund is automatically issued — with confirmation sent to the customer and logged in your ERP.

Financial Reconciliation

Retrieve payment intents and invoice data to feed your accounting workflows. Automate revenue recognition, VAT calculation, and monthly billing summaries without manual Stripe exports.

Use Cases

SaaS Onboarding to First Payment

New user signs up → create Stripe customer → create subscription → send welcome email with billing confirmation → provision account. The entire sequence in a single triggered workflow.

Invoice-to-Cash

Generate a Stripe invoice, finalize it, and email it to the customer. When payment is received (via webhook), update CRM and trigger fulfillment — fully automated accounts receivable.

Churn Prevention Refund

When a high-value customer requests cancellation, route to a CS agent approval workflow. On approval, issue a partial refund, update the subscription, and send a retention offer — all automated.

B2B Usage Billing

At end of month, collect usage metrics, calculate the charge amount, create a Stripe invoice with itemized line items, and send to the customer — replacing manual billing operations.

Credential Setup

The Stripe node requires a credential stored in BizFirst Credentials Manager with the following fields:

FieldDescription
Secret KeyYour Stripe Secret Key (sk_live_... for production, sk_test_... for test). Found in Stripe Dashboard → Developers → API keys.
ModeSelect live or test to automatically validate that the key matches the expected mode.
Webhook SecretOptional. Required if you verify Stripe webhook signatures in BizFirst (recommended for production WebhookTrigger workflows receiving Stripe events).

Idempotency Key Best Practices

Stripe supports idempotency keys on all POST operations. When a workflow retries due to a timeout, the same idempotency key ensures Stripe returns the original result rather than creating a duplicate charge, customer, or subscription. Always set idempotency_key on:

  • create_payment_intent — use the order ID
  • create_customer — use the internal user ID
  • create_subscription — use the user ID + plan ID
  • create_refund — use the support ticket ID or refund request ID

Idempotency keys are valid for 24 hours on Stripe. After that, the same key can be reused for a new operation.

Webhook integration tip: Most Stripe operations are asynchronous from a business perspective. Payment confirmation, subscription status changes, and invoice payment events arrive as Stripe webhook events. Use a WebhookTrigger node configured with your Stripe webhook endpoint to handle events like payment_intent.succeeded, invoice.payment_failed, and customer.subscription.deleted — and trigger the appropriate downstream workflow steps.

Key Stripe Webhook Events to Handle in BizFirst

Stripe EventMeaningRecommended BizFirst Action

payment_intent.payment_failedPayment attempt failedNotify customer, log failure, trigger retry workflow
invoice.payment_succeededSubscription invoice paidUpdate CRM, send receipt, renew access
invoice.payment_failedSubscription invoice payment failedSend dunning email, restrict access after grace period
customer.subscription.createdNew subscription startedProvision access, send welcome email
customer.subscription.deletedSubscription cancelledRevoke access, send offboarding email, update CRM
customer.subscription.updatedPlan changed, quantity changed, etc.Adjust provisioned access, send confirmation
refund.createdRefund issuedLog to ERP, send refund confirmation to customer

Set up a single BizFirst WebhookTrigger endpoint for all Stripe events. Use a Switch node on the type field of the Stripe webhook payload to route each event type to its own handling workflow branch.
