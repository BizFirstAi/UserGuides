Common Error Types

Error TypeCause
card_errorCard declined, expired, insufficient funds. Recoverable — prompt customer to use a different card.
invalid_request_errorInvalid parameter (wrong amount format, invalid customer ID). Fix the configuration.
authentication_errorInvalid API key. Check the credential in BizFirst Credentials Manager.
rate_limit_errorToo many requests. Add a Delay node and retry.
api_errorStripe server error. Safe to retry with exponential backoff.
Accessing output in downstream nodes: After create_payment_intent, use $ctx.id for the PaymentIntent ID and $ctx.client_secret for the frontend confirmation key. After create_customer, use $ctx.id as the customer ID for subsequent operations.
