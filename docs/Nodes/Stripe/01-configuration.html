Configuration

All properties for each Stripe operation — payment intents, customers, subscriptions, refunds, and invoices.

Common Properties (All Operations)

PropertyTypeDescription
credential_id RequiredstringID of a BizFirst Stripe credential containing the Secret Key (sk_live_... or sk_test_...).
operation RequiredstringOne of: create_payment_intent, retrieve_payment_intent, create_customer, retrieve_customer, create_subscription, cancel_subscription, create_refund, create_invoice.
idempotency_key OptionalstringUnique key to prevent duplicate operations on retry. Stripe returns the same result for repeated requests with the same key. Use a workflow run ID or order ID.

create_payment_intent

PropertyTypeDescription
amount RequiredintegerAmount in the smallest currency unit (cents for USD, pence for GBP). e.g. 5000 = $50.00.
currency Requiredstring3-letter ISO currency code: usd, gbp, eur, aud, etc.
customer OptionalstringStripe Customer ID to associate with this payment intent.
description OptionalstringInternal description for the payment (visible in Stripe dashboard).
metadata OptionalobjectCustom key-value pairs attached to the PaymentIntent for business reference (order ID, invoice number, etc.).
capture_method Optionalstringautomatic (default, capture on confirmation) or manual (authorize now, capture later).
payment_method_types OptionalarrayAllowed payment methods: ["card"], ["card", "sepa_debit"], etc. Default: ["card"].

create_customer

PropertyTypeDescription
email RequiredstringCustomer email address. Used for receipts and subscription communications.
name OptionalstringFull name of the customer.
phone OptionalstringCustomer phone number (E.164 format).
description OptionalstringInternal note about this customer.
metadata OptionalobjectBusiness-specific identifiers: {"internal_id": "usr_123", "plan": "enterprise"}.

create_subscription

PropertyTypeDescription
customer RequiredstringStripe Customer ID to subscribe.
items RequiredarrayArray of price objects: [{"price": "price_abc123"}]. The price ID references a Stripe Price created in the dashboard.
trial_period_days OptionalintegerNumber of free trial days before billing begins.
cancel_at_period_end OptionalbooleanIf true, subscription cancels at the end of the current billing period instead of immediately.
metadata OptionalobjectCustom metadata for the subscription record.

cancel_subscription

PropertyTypeDescription
subscription_id RequiredstringStripe Subscription ID to cancel (format: sub_...).
cancel_at_period_end Optionalbooleantrue — cancel at end of period (grace period). false — cancel immediately (default).

create_refund

PropertyTypeDescription
payment_intent RequiredstringStripe PaymentIntent ID to refund (format: pi_...).
amount OptionalintegerAmount to refund in smallest currency units. Omit for a full refund.
reason Optionalstringduplicate, fraudulent, or requested_by_customer.
metadata OptionalobjectReference data for the refund (ticket ID, agent name, etc.).

create_invoice

PropertyTypeDescription
customer RequiredstringStripe Customer ID to invoice.
auto_advance Optionalbooleantrue — automatically finalize and send the invoice. false — leave as draft for manual review. Default: true.
collection_method Optionalstringcharge_automatically (charge saved payment method) or send_invoice (email invoice for manual payment). Default: charge_automatically.
days_until_due OptionalintegerFor send_invoice: number of days until the invoice is due.
description OptionalstringDescription shown on the invoice PDF.
metadata OptionalobjectCustom metadata attached to the invoice object.
Live vs. Test mode: Store separate sk_test_... and sk_live_... credentials in BizFirst Credentials Manager. Always validate workflows with the test credential before switching to live. Live keys process real financial transactions.

retrieve_customer

PropertyTypeDescription
customer_id RequiredstringStripe Customer ID (cus_...) to retrieve. Returns the full customer object including attached payment methods and metadata.

retrieve_payment_intent

PropertyTypeDescription
payment_intent_id RequiredstringStripe PaymentIntent ID (pi_...) to retrieve. Use this in webhook-triggered workflows to confirm the current status of a payment before triggering fulfillment.

Full Example Configurations

create_payment_intent — Full Configuration

{
  "operation": "create_payment_intent",
  "credential_id": "stripe-live",
  "amount": 4900,
  "currency": "usd",
  "customer": "{@ $var.stripeCustomerId }",
  "description": "Order #{@ $var.orderId } — {@ $var.productName }",
  "metadata": {
    "order_id": "{@ $var.orderId }",
    "product_sku": "{@ $var.productSku }",
    "workflow_run_id": "{@ $workflow.runId }"
  },
  "payment_method_types": ["card"],
  "capture_method": "automatic",
  "idempotency_key": "{@ $var.orderId }"
}

create_subscription — Full Configuration

{
  "operation": "create_subscription",
  "credential_id": "stripe-live",
  "customer": "{@ $node.createCustomer.id }",
  "items": [
    { "price": "{@ $var.priceId }" }
  ],
  "trial_period_days": 14,
  "cancel_at_period_end": false,
  "metadata": {
    "user_id": "{@ $var.userId }",
    "plan_name": "{@ $var.planName }",
    "utm_source": "{@ $var.utmSource }"
  },
  "idempotency_key": "{@ $var.userId }-{@ $var.priceId }"
}

create_refund — Full Configuration

{
  "operation": "create_refund",
  "credential_id": "stripe-live",
  "payment_intent": "{@ $var.paymentIntentId }",
  "amount": "{@ $var.refundAmount }",
  "reason": "requested_by_customer",
  "metadata": {
    "ticket_id": "{@ $var.supportTicketId }",
    "agent_id": "{@ $var.agentId }",
    "refund_reason": "{@ $var.customerReason }"
  },
  "idempotency_key": "refund-{@ $var.supportTicketId }"
}
Amount units: Stripe always works in the smallest currency unit. For USD and EUR: 100 cents = $1.00. For JPY and KRW (zero-decimal currencies): 1 unit = ¥1. Always store and pass amounts in the smallest unit. A common mistake is passing 49.00 instead of 4900 for a $49 charge — this would result in a $0.49 charge.
