Examples

Five real-world Gmail node configurations covering send, list, label management, thread reply, and attachment retrieval.

Example 1: Send a Customer Support Reply Within a Thread

Scenario: A support agent has resolved a ticket. Reply to the customer's original email within the same thread with the resolution details.

{
  "credential_id": "google-oauth-support-inbox",
  "operation": "send",
  "to": "{{ vars.customer_email }}",
  "subject": "Re: {{ vars.original_subject }}",
  "thread_id": "{{ vars.customer_thread_id }}",
  "html_body": "<p>Hi {{ vars.customer_first_name }},</p><p>Thank you for contacting support. Your issue has been resolved:</p><blockquote>{{ vars.resolution_summary }}</blockquote><p>If you need further assistance, please reply to this email.</p><p>Best regards,<br>{{ vars.agent_name }}<br>{{ env.COMPANY_NAME }} Support</p>",
  "body": "Hi {{ vars.customer_first_name }}, your issue has been resolved: {{ vars.resolution_summary }}. Reply if you need more help."
}
Expected outcome: Reply appears in the customer's original email thread, maintaining full conversation context. The thread_id from the original inbound email is passed as a workflow variable.

Example 2: List Unread Invoice Emails for Processing

Scenario: Every hour, fetch unread emails in the accounts inbox that have PDF attachments and contain "invoice" in the subject.

{
  "credential_id": "google-oauth-accounts-inbox",
  "operation": "list",
  "query": "is:unread subject:invoice has:attachment label:inbox newer_than:1d",
  "max_results": 50,
  "label_ids": ["INBOX", "UNREAD"]
}
Expected outcome: Returns up to 50 unread invoice emails. The messages array is then iterated by a Loop node. Each iteration fetches the full message and processes the PDF attachment through an OCR/AI extraction workflow.

Example 3: Archive and Label a Processed Email

Scenario: After successfully processing an invoice email, mark it as read, remove it from the inbox, and apply a custom "Invoice-Processed" label.

{
  "credential_id": "google-oauth-accounts-inbox",
  "operation": "modify",
  "message_id": "{{ vars.email_message_id }}",
  "add_labels": ["Label_5823947123"],
  "remove_labels": ["INBOX", "UNREAD"]
}
Replace Label_5823947123 with your custom Gmail label ID. Retrieve label IDs from the Gmail Labels API or via the Gmail settings UI.

Example 4: Fetch Full Email Content Including Attachments

Scenario: Retrieve the complete body and all attachments from a specific email for downstream AI document processing.

{
  "credential_id": "google-oauth-accounts-inbox",
  "operation": "get",
  "message_id": "{{ vars.target_message_id }}",
  "format": "full",
  "fetch_attachments": true
}
Expected outcome: Full message object returned including body_text, body_html, sender headers, and attachment data (base64-encoded). Attachments are passed to the FlowRag or AI processing node for extraction.

Example 5: Send Newsletter to Subscriber List

Scenario: A Loop node iterates over a subscriber array. For each subscriber, send a personalised newsletter email from the marketing Gmail account.

{
  "credential_id": "google-oauth-marketing-account",
  "operation": "send",
  "to": "{{ vars.subscriber_email }}",
  "subject": "{{ env.NEWSLETTER_SUBJECT }} — {{ vars.newsletter_month }}",
  "from_name": "{{ env.COMPANY_NAME }} Newsletter",
  "html_body": "<p>Hi {{ vars.subscriber_first_name }},</p>{{ vars.newsletter_html_content }}<p><small><a href='{{ vars.unsubscribe_url }}'>Unsubscribe</a></small></p>",
  "body": "Hi {{ vars.subscriber_first_name }}, {{ vars.newsletter_text_content }}. Unsubscribe: {{ vars.unsubscribe_url }}"
}
Expected outcome: Each subscriber receives a personalised email. The unsubscribe link is unique per subscriber (generated upstream). Gmail's daily sending limit (typically 500/day for Workspace) applies — for larger lists use the Email (SMTP) node with a bulk relay.
