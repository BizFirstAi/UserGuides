Configuration

Full property reference for all five Gmail node operations: send, list, get, delete, and modify.

Node Identity

PropertyValue
Resourcegmail
Operationssend, list, get, delete, modify
Auth TypeOAuth2 (Google)
Required Scopehttps://mail.google.com/
Output Portssuccess, error

Common Properties (All Operations)

PropertyRequiredDescription
credential_id Required Reference to a Google OAuth2 credential stored in BizFirst Credentials Manager. Must have the Gmail full-access scope.
operation Required The Gmail operation to perform. One of: send, list, get, delete, modify.

Send Operation

PropertyRequiredDescription
toRequiredRecipient email address(es). String or array. Supports expressions.
ccOptionalCC recipients. String or array.
bccOptionalBCC recipients. String or array.
subjectRequiredEmail subject line. Supports expressions.
bodyOptionalPlain text body. Used as fallback when html_body is provided.
html_bodyOptionalHTML email body. Supports expressions and Liquid templates.
thread_idOptionalGmail thread ID to reply within. When set, the email is added to the existing conversation thread rather than starting a new one.
attachmentsOptionalArray of attachment objects. Each has filename, content_type, source, and data fields. Identical schema to the SMTP node.

List Operation

PropertyRequiredDescription
query Optional Gmail search query string using the same syntax as the Gmail search bar. Examples: is:unread from:invoices@supplier.com, subject:invoice has:attachment newer_than:7d, label:support-queue is:unread.
max_results Optional Maximum number of messages to return. Default: 10. Maximum: 500.
label_ids Optional Array of label IDs to filter by. System labels: INBOX, UNREAD, SENT, TRASH, SPAM. Also accepts custom label IDs.
page_token Optional Pagination token from a previous list response to retrieve the next page of results.
include_spam_trash Optional Whether to include messages from SPAM and TRASH in results. Default: false.

Get Operation

PropertyRequiredDescription
message_id Required The Gmail message ID to retrieve. Typically obtained from a prior list operation output. Example: {{ vars.message_id }}.
format Optional Response format. One of full (all headers and body), metadata (headers only), minimal (IDs only), raw (RFC 2822 format). Default: full.
fetch_attachments Optional When true, automatically fetch and include attachment data in the output. Default: false. Large attachments can significantly increase execution time.

Delete Operation

PropertyRequiredDescription
message_id Required The Gmail message ID to permanently delete. This operation bypasses Trash — the message is immediately and irreversibly removed.

Modify Operation (Labels)

PropertyRequiredDescription
message_id Required The Gmail message ID to modify.
add_labels Optional Array of label IDs to add to the message. To mark as read: remove UNREAD. To archive: remove INBOX. To apply a custom label, use the label's ID (retrieve via Gmail Labels API).
remove_labels Optional Array of label IDs to remove from the message.
Common label operations:
Mark as read: remove_labels: ["UNREAD"]
Archive (remove from inbox): remove_labels: ["INBOX"]
Move to trash: add_labels: ["TRASH"]
Star a message: add_labels: ["STARRED"]
Rate limits: The Gmail API has a quota of 250 quota units per user per second. The send operation costs 100 units, list costs 5, and get costs 5. Design high-volume workflows with rate limiting in mind.
