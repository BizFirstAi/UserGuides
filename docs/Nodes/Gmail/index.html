Gmail

Send, read, manage, and organise Gmail messages via the Gmail API with full OAuth2 authentication.

What this node does: The Gmail node provides complete Gmail API integration — send emails, list and read messages, manage labels, delete messages, and handle threaded conversations. Unlike SMTP, Gmail API integration provides full mailbox management capabilities, read receipts, and access to Gmail's powerful search syntax. Authentication is handled via OAuth2 using a stored Google credential.

Supported Operations

OperationDescription
sendSend a new email message. Supports HTML, attachments, CC/BCC, and thread replies.
listList messages matching a Gmail search query. Returns message IDs, thread IDs, and snippets.
getRetrieve the full content of a specific message by its ID, including headers, body, and attachments.
deletePermanently delete a message from the mailbox.
modifyAdd or remove labels from a message (e.g. mark as read, archive, apply custom labels).

Key Capabilities

  • OAuth2 authentication via stored Google credential — no SMTP configuration required
  • Full Gmail search query support in list operation (same syntax as Gmail search bar)
  • Thread-aware sending — reply within an existing conversation thread using thread_id
  • Label management — apply, remove, and read custom and system Gmail labels
  • Attachment access — retrieve file attachments from received emails for downstream processing
  • Pagination support for list operations via next_page_token
  • Dual output ports on all operations: success and error

Business Benefits

Inbox-Driven Process Automation

Transform your Gmail inbox into a workflow trigger source. Use the list operation to find unread emails matching specific criteria, then route each one through downstream processing — extracting data, creating records, or triggering responses automatically.

Conversational Thread Management

Maintain coherent email conversations by replying within the same thread. Support tickets, procurement requests, and approval chains can all be handled as threaded Gmail conversations, maintaining full context for all parties.

Automated Email Triage

Apply Gmail labels automatically based on workflow logic — mark invoices as "Processed", tag support emails with customer tier labels, or archive newsletters after extracting relevant content. Keep inboxes clean and organised at scale.

Document Extraction from Email Attachments

Fetch email attachments (invoices, purchase orders, contracts) in workflows and pass them to AI processing nodes for data extraction. Eliminate manual document collection and entry entirely.

Use Cases

Send Newsletters and Bulk Communications

Send personalised newsletters to a list of subscribers by looping through contact records and using the Gmail send operation for each. Gmail's per-day sending limits apply — for high-volume sends, consider the Email (SMTP) node with a bulk relay instead.

Auto-Reply to Support Emails

List unread emails in the support inbox, extract the sender and subject, generate an AI-driven response, and send the reply within the same thread — all automatically, within minutes of the original email's arrival.

Archive Processed Emails

After extracting data from an email (e.g. an order confirmation from a supplier), use the modify operation to apply an "Archived" label and remove it from the Inbox, keeping the mailbox focused on actionable items only.

Label Incoming Invoices

When an email containing an invoice attachment arrives, apply a custom "Invoice-Received" label and set a "To-Process" label. A downstream workflow triggered on this label can then route the invoice through the AP processing pipeline.

Fetch Unread Support Emails for CRM

Every 15 minutes, list unread emails in the support inbox. For each email, create a support ticket in the CRM, assign it to the appropriate team based on subject keywords, and mark the email as read with a "Ticket-Created" label.

Thread-Based Approval Management

When an approval request is sent via the Gmail node within a thread, monitor replies in that thread to detect approval or rejection keywords. Resume the paused workflow based on the email reply content — a lightweight, email-native HIL pattern.
