Configuration

Full property reference for all four WhatsApp node operations.

Node Identity

PropertyValue
Operationssend, sendTemplate, getMedia, getStatus
API ProviderMeta WhatsApp Business API (Cloud or On-Premise)
Output Portssuccess, error

Common Properties

PropertyRequiredDescription
credential_idRequiredReference to a WhatsApp Business API credential. Must contain phone_number_id and access_token.
operationRequiredThe operation to perform.

send Operation

PropertyRequiredDescription
phone_numberRequiredRecipient's phone number in E.164 format including country code (e.g. +14155552671 or {{ vars.customer_phone }}). Do not include spaces, dashes, or parentheses.
message_typeRequiredType of message to send. One of: text, image, document, audio.
message_textOptionalThe text content for text type messages. Supports BizFirst expressions. Maximum 4096 characters.
media_urlOptionalPublicly accessible URL of the media file for image, document, or audio message types. Must be HTTPS.
media_captionOptionalOptional caption text to accompany image or document messages. Supports expressions.
media_filenameOptionalFilename to display with document attachments (e.g. Invoice_{{ vars.invoice_number }}.pdf).

sendTemplate Operation

PropertyRequiredDescription
phone_numberRequiredRecipient phone number in E.164 format. Supports expressions.
template_nameRequiredName of the pre-approved Meta message template (e.g. order_shipped_v2).
template_languageRequiredLanguage code for the template (e.g. en_US, en_GB, es_MX). Must match the template's approved language.
parametersOptionalArray of variable values to inject into the template placeholders in order. Each element is a string. Example: ["{{ vars.customer_name }}", "{{ vars.order_number }}", "{{ vars.tracking_url }}"].
header_media_urlOptionalFor templates with media headers: the URL of the header image or document.

getMedia Operation

PropertyRequiredDescription
media_idRequiredThe media ID from an incoming WhatsApp message webhook payload. Used to download the actual media file.

getStatus Operation

PropertyRequiredDescription
message_idRequiredThe message ID returned by a previous send or sendTemplate operation.

Template Parameter Example

{
  "operation": "sendTemplate",
  "phone_number": "{{ vars.customer_phone }}",
  "template_name": "order_shipped_v2",
  "template_language": "en_US",
  "parameters": [
    "{{ vars.customer_first_name }}",
    "{{ vars.order_number }}",
    "{{ vars.carrier_name }}",
    "{{ vars.tracking_number }}",
    "{{ vars.tracking_url }}"
  ]
}
Template parameters are positional. The first element maps to {{1}} in the template, the second to {{2}}, and so on. Ensure the parameter count matches the template definition exactly.
Phone number format: Always use E.164 format including the country code and plus sign. Incorrect formats will result in delivery failures. Use {{ format_phone(vars.phone, "E164") }} to normalise phone numbers from various input formats.
