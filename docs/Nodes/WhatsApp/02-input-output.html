Input & Output

Output ports and data schemas for all WhatsApp node operations.

Output Ports

PortTriggerDescription
successAPI accepted the messageThe WhatsApp Business API accepted the message for delivery. Note: acceptance does not guarantee delivery — the recipient's device must be online.
errorAPI errorThe request was rejected — invalid phone number, invalid template name, expired token, rate limit exceeded, or network failure.

send / sendTemplate Output

FieldTypeDescription
message_idstringWhatsApp message ID (format: wamid.xxx). Use with getStatus to track delivery.
statusstringInitial message status. Typically accepted immediately after API call. Final delivery statuses (sent, delivered, read) arrive via webhook.
timestampstringISO 8601 timestamp of when the message was accepted by the API.
phone_numberstringThe recipient phone number as used in the API call.

getStatus Output

FieldTypeDescription
message_idstringThe message ID queried.
statusstringCurrent delivery status: sent, delivered, read, failed.
timestampstringTimestamp of the last status update.
error_codeintegerWhatsApp error code if status is failed. Reference Meta's error code documentation for details.

getMedia Output

FieldTypeDescription
media_datastringBase64-encoded media file content. Decode and process downstream (e.g. pass to AI extraction node).
mime_typestringMIME type of the media (e.g. image/jpeg, application/pdf).
file_sizeintegerSize of the media file in bytes.
