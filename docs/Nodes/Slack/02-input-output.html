Input & Output

Output ports and data schemas for all Slack node operations, including the sendAndWait HIL response payload.

Output Ports

PortOperationsDescription
successAll operationsSlack API accepted the request and the operation completed successfully.
errorAll operationsSlack API returned an error, the token was invalid, or the operation timed out.
responsesendAndWait onlyA user interacted with the interactive message. The workflow resumes on this port with the user's response payload.

message.send / message.update Output

FieldTypeDescription
okbooleantrue on success.
channelstringChannel ID the message was posted to.
tsstringTimestamp of the sent message. Used as identifier for updates, thread replies, and reactions.
message_textstringThe text content of the sent message as confirmed by Slack.

sendAndWait Response Port Output

FieldTypeDescription
action_idstringThe action_id of the button the user clicked (e.g. approve_purchase, reject_purchase).
action_valuestringOptional value associated with the button element.
user_idstringSlack user ID of the person who clicked the button.
user_namestringSlack username of the responder.
response_timestringISO 8601 timestamp of when the user responded.
original_message_tsstringTimestamp of the original interactive message that was responded to.

channel.create Output

FieldTypeDescription
channel_idstringID of the newly created channel.
channel_namestringName of the channel (normalised to lowercase).
createdintegerUnix timestamp of channel creation.
is_privatebooleanWhether the channel is private.

file.upload Output

FieldTypeDescription
file_idstringSlack file ID for the uploaded file.
file_urlstringURL to the file within Slack (requires authentication to access).
filenamestringFilename as confirmed by Slack.
sizeintegerFile size in bytes.

Example: message.send Output

{
  "ok": true,
  "channel": "C012AB3CD",
  "ts": "1731578400.000200",
  "message_text": "Deployment to production completed successfully. Build: v2.4.1"
}

{
  "action_id": "approve_purchase",
  "action_value": "approved",
  "user_id": "U023BECGF",
  "user_name": "sarah.chen",
  "response_time": "2025-11-14T11:32:07Z",
  "original_message_ts": "1731578400.000200"
}
Routing on sendAndWait response: Use a Switch node downstream of the response port. Switch on {{ nodes.slackApproval.output.action_id }} to route to different workflow branches for each possible button response.
