Configuration

Complete property reference for all Slack node resources and operations.

Node Identity

PropertyValue
Resource Typesmessage, channel, file, reaction, user, userGroup
Auth TypeSlack Bot Token (xoxb-...)
Output Portssuccess, error, response (sendAndWait only)

Common Properties

PropertyRequiredDescription
credential_idRequiredSlack Bot Token credential ID stored in BizFirst Credentials Manager.
resourceRequiredThe Slack resource type. One of: message, channel, file, reaction, user, userGroup.
operationRequiredThe operation to perform on the resource (see tables below).

message.send Properties

PropertyRequiredDescription
channelRequiredChannel ID or name (e.g. #general, C012AB3CD), or a user ID for DMs.
textOptionalPlain text message content. Used as fallback when blocks are provided, and for notifications in clients that don't render blocks.
blocksOptionalRich Block Kit layout as a JSON array. Overrides plain text rendering. See Block Kit reference below.
thread_tsOptionalTimestamp of the parent message to reply in a thread. Format: 1609459200.000100.
usernameOptionalOverride the bot's display name for this message.
icon_emojiOptionalOverride the bot's avatar with an emoji (e.g. :robot_face:).
unfurl_linksOptionalWhether to unfurl URLs in the message. Default: true.

message.sendAndWait Properties

In addition to all message.send properties, sendAndWait requires:

PropertyRequiredDescription
wait_for_responseRequiredMust be true. Tells the engine to suspend the workflow execution thread pending a Slack interaction callback.
resume_actionsRequiredArray of button action IDs that will trigger workflow resumption. When a user clicks a button whose action_id matches one of these values, the workflow resumes on the response output port.
timeout_hoursOptionalMaximum time in hours to wait for a user response before timing out. On timeout, the workflow routes to the error port. Default: 72 hours.

channel Operations

OperationKey Properties
createname (required), is_private (bool, default false)
archivechannel (channel ID, required)
invitechannel (required), users (array of user IDs, required)
joinchannel (channel ID, required)
renamechannel (required), name (new name, required)

file.upload Properties

PropertyRequiredDescription
channelsRequiredArray of channel IDs where the file will be shared.
filenameRequiredFilename to display in Slack.
file_dataRequiredFile content as base64 string or a workflow variable reference.
titleOptionalOptional title displayed above the file in Slack.
initial_commentOptionalOptional text message posted alongside the file upload.

Block Kit JSON Structure

Block Kit is Slack's layout framework for rich messages. Below is the structure for a complete approval message with buttons:

[
  {
    "type": "header",
    "text": {
      "type": "plain_text",
      "text": ":bell: Purchase Request Approval Required"
    }
  },
  {
    "type": "section",
    "fields": [
      { "type": "mrkdwn", "text": "*Requested by:*\n{{ vars.requester_name }}" },
      { "type": "mrkdwn", "text": "*Amount:*\n{{ vars.pr_amount_formatted }}" },
      { "type": "mrkdwn", "text": "*Vendor:*\n{{ vars.vendor_name }}" },
      { "type": "mrkdwn", "text": "*Request #:*\n{{ vars.pr_number }}" }
    ]
  },
  {
    "type": "section",
    "text": {
      "type": "mrkdwn",
      "text": "*Justification:* {{ vars.justification }}"
    }
  },
  {
    "type": "actions",
    "elements": [
      {
        "type": "button",
        "text": { "type": "plain_text", "text": "Approve" },
        "style": "primary",
        "action_id": "approve_purchase"
      },
      {
        "type": "button",
        "text": { "type": "plain_text", "text": "Reject" },
        "style": "danger",
        "action_id": "reject_purchase"
      }
    ]
  }
]
sendAndWait and interactive components: For sendAndWait to work, your Slack App must have Interactive Components enabled, and the Request URL must be set to your BizFirst workflow engine's Slack callback endpoint. Contact your BizFirst administrator for the correct endpoint URL.
