GitHub

Automate GitHub repository and project management — create issues, open pull requests, post comments, and drive DevOps pipelines from workflows.

What this node does: The GitHub node integrates with the GitHub REST API to automate repository operations. Create issues automatically from monitoring alerts, open pull requests when workflow steps complete, comment on issues with deployment or processing results, list and query open issues, and integrate GitHub into end-to-end DevOps automation pipelines — all driven by workflow context via BizFirst expressions.

Supported Operations

OperationDescription
createIssueCreate a new issue in a repository with title, body, labels, assignees, and milestone.
createPullRequestOpen a pull request from a head branch to a base branch with full metadata.
commentOnIssuePost a comment on an existing issue or pull request by number.
mergeAsyncMerge a pull request using merge, squash, or rebase strategy.
getIssueRetrieve full details of a specific issue or pull request by number.
listIssuesList issues in a repository with filtering by state, labels, assignee, and milestone.

Key Capabilities

  • Full issue lifecycle management — create, comment, read, and list with rich filtering
  • Pull request automation — open PRs, post review comments, and trigger merges
  • Dynamic labels, assignees, and milestones from workflow expressions
  • GitHub Personal Access Token or GitHub App authentication
  • Supports GitHub.com and GitHub Enterprise Server
  • Dual output ports: success and error

Business Benefits

Automated Bug Reporting from Monitoring

When application monitoring detects an error exceeding a threshold, automatically create a GitHub issue with the full error context, stack trace, environment details, and a link to the monitoring dashboard — giving developers everything they need immediately.

DevOps Pipeline Integration

Trigger GitHub pull requests, issue status updates, and merge operations as part of larger deployment workflows. Connect your release management process, change approval flows, and environment promotion steps directly to GitHub actions.

Sprint Tracking Automation

When business processes complete milestones (e.g. client sign-off, QA approval), automatically close GitHub issues, update labels, or post progress comments — keeping technical and business tracking in sync without manual updates.

Use Cases

Auto-Create Bug Reports from Monitoring Alerts

When Datadog/PagerDuty triggers a workflow for an application error, create a GitHub issue with severity label, assignee, and full error details automatically.

Create PR on Workflow Completion

After an automated code generation or configuration update step completes, open a pull request to the main branch with a descriptive body summarising the changes made.

Post Deployment Status Comments

After deploying to staging or production, comment on the related PR with the deployment result, environment URL, and test coverage summary.

Track Sprint Items via Workflow

When a business process milestone is reached, close the corresponding GitHub issue and update its label to "Done" — keeping Scrum boards in sync with business processes.

Auto-Assign Issues

When new issues are created in a repository, a webhook-triggered BizFirst workflow analyses the issue content and auto-assigns it to the correct team member based on keyword matching and team schedules.
