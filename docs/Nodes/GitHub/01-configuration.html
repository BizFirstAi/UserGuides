Configuration

Complete property reference for all six GitHub node operations.

Common Properties

PropertyRequiredDescription
credential_idRequiredGitHub PAT or GitHub App credential reference stored in BizFirst Credentials Manager.
operationRequiredThe GitHub operation to perform.
ownerRequiredRepository owner — a GitHub username or organisation name. Supports expressions.
repoRequiredRepository name (without owner). Supports expressions.

createIssue Properties

PropertyRequiredDescription
titleRequiredIssue title. Supports expressions for dynamic content.
bodyOptionalIssue body in GitHub Markdown. Supports expressions. Useful for embedding error details, links, and structured information.
labelsOptionalArray of label names to apply. Labels must exist in the repository. Example: ["bug", "high-priority", "{{ vars.env_label }}"].
assigneesOptionalArray of GitHub usernames to assign. Example: ["{{ vars.on_call_github_user }}"].
milestoneOptionalMilestone number (integer) to associate the issue with.

createPullRequest Properties

PropertyRequiredDescription
titleRequiredPull request title. Supports expressions.
headRequiredThe head branch with changes (e.g. feature/auto-config-update or {{ vars.branch_name }}).
baseRequiredThe base branch to merge into (e.g. main, develop).
bodyOptionalPull request description in Markdown. Include change summary, test results, and related issue references.
draftOptionalCreate the PR as a draft. Default: false.
reviewersOptionalArray of GitHub usernames to request reviews from.

commentOnIssue Properties

PropertyRequiredDescription
issue_numberRequiredThe issue or PR number to comment on. Supports expressions.
bodyRequiredComment body in GitHub Markdown. Supports BizFirst expressions for dynamic, data-driven comments.

mergeAsync Properties

PropertyRequiredDescription
pull_numberRequiredThe pull request number to merge.
merge_methodOptionalMerge strategy. One of merge, squash, rebase. Default: merge.
commit_messageOptionalCustom commit message for the merge commit. Supports expressions.

listIssues Properties

PropertyRequiredDescription
stateOptionalFilter by issue state. One of open, closed, all. Default: open.
labelsOptionalComma-separated list of label names to filter by.
assigneeOptionalFilter by assignee username. Use * for any assignee.
per_pageOptionalResults per page. Default: 30. Max: 100.
sortOptionalSort field. One of created, updated, comments. Default: created.
GitHub Enterprise: For GitHub Enterprise Server, set api_base_url in the credential to your GHES API endpoint (e.g. https://github.yourcompany.com/api/v3).
