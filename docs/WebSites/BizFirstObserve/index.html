Observability

BizFirst Observe

Enterprise observability for the BizFirst ecosystem — Loki (logs), Prometheus (metrics), Tempo (traces), and Grafana (dashboards) via OpenTelemetry.

Physical Architecture

OTel Collector, component topology, data flows, Alertmanager, Node Exporter.

Default Architecture

Loki + Prometheus + Tempo + Grafana stack with 10 pre-built dashboards.

Loki — Logs

Label strategy, LogQL queries, structured JSON log schema, chunk storage.

Prometheus — Metrics

Metric types, BizFirst metrics catalog, scrape config, AlertManager rules.

Tempo — Traces

Distributed tracing, OTLP ingestion, BizFirst span structure, exemplars.

Grafana — Dashboards

Data source setup, pre-built dashboards, variables, alert panels.

Setup Guide

End-to-end: Docker Compose, OTel Collector config, service OTLP settings, verification.

Data Retention

Retention policies per component, S3 archival, backup, cost optimization.

Sensitive Data

PII scrubbing, OTel redaction processor, GDPR compliance, LBAC in Loki.

Using the Tools

Grafana Explore, LogQL search, trace correlation, alert management, annotations.

Enterprise Options

HA deployment, microservices mode, clustering, Enterprise Loki/Grafana.

PaaS Options

Grafana Cloud, Azure Monitor, Datadog, AWS — OTel exporter swapping.

TEE

Observability in Trusted Execution Environments — constraints, encrypted shipping.
