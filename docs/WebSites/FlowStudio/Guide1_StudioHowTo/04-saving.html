Saving Your Work

How to save, auto-save, and manage versions of your workflows in Flow Studio.

Save Mechanics

As you edit a workflow in the designer, your changes are held in memory. To persist them, you must explicitly save.

Manual Save

Click the Save button in the toolbar, or use the keyboard shortcut Ctrl+S (Windows) / Cmd+S (Mac).

Unsaved Changes Indicator

The title bar shows a dot indicator when the current thread has unsaved changes. This helps you avoid accidentally losing work.

Auto-Save

Flow Studio can be configured to automatically save your work at regular intervals (typically every 30 seconds). Check your project settings to enable auto-save.

Draft vs Published

Draft State

When you save a workflow, it remains in draft state. Draft workflows are not executable by triggers — they are safe to edit without affecting running processes.

Publishing

To make a workflow available for execution, you must explicitly publish it. Publishing creates a new version that can be triggered and executed.

Version History

Each time you publish, a new version is created. You can view and roll back to previous versions from the process details panel.

Lock & Unlock

When you open a ProcessThread to edit, it is automatically locked to you. This prevents other team members from editing the same thread simultaneously.

When you close the designer or navigate away, the thread is automatically unlocked, making it available for others to edit.

Handling Conflicts

If another user edits a thread while you have it open, Flow Studio will alert you when you try to save. You can:

  • Refresh — Discard your changes and reload the latest version
  • Merge — Attempt an automatic merge (if changes are in different areas)
  • Conflict Resolver — Manually resolve line-by-line conflicts

Best Practices

  • Save frequently to avoid losing work
  • Use descriptive names for processthreads so team members understand what you're working on
  • Publish only when you're confident the workflow is ready for execution
  • Review version history before rolling back to ensure you're reverting the right changes
