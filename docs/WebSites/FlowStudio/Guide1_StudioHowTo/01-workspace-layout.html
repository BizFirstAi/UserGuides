Workspace Layout

Overview of the Flow Studio workspace — the dashboard navigation hierarchy, canvas designer, and the project/process/thread model.

The Flow Studio Workspace

Flow Studio is organized around three core concepts that form a hierarchy:

  • Project — The top-level container for all your workflows and data.
  • Process — A workflow definition (a versioned blueprint of nodes and edges).
  • ProcessThread — A concrete instance of a Process that you edit in the canvas designer.

Main Navigation Areas

Dashboard (Three-Panel Navigator)

When you first log in, you see the unified dashboard with three columns:

PanelPurpose
LeftProject browser — lists all projects you have access to
CentreProcess list — shows workflow definitions in the selected project
RightProcessThread list — shows instances for the selected process

Canvas Designer

When you open a ProcessThread, you enter the canvas designer — a visual editor for building workflows with nodes, edges, and properties.

Key Concepts

Projects

A project is a multi-tenant container that holds all your workflows, team members, and access control settings. Every project belongs to a single tenant.

Processes (Workflow Definitions)

A Process is a versioned blueprint. It defines the node graph, but is never executed directly. Instead, you create ProcessThreads from it.

ProcessThreads (Workflow Instances)

A ProcessThread is an editable snapshot of a Process. This is what you open in the designer and execute. Each thread tracks its own execution history and state.

Workflow

  1. Start in the dashboard
  2. Select or create a project
  3. Select or create a process (workflow definition)
  4. Create or open a processthread
  5. Open the designer to edit
  6. Save and publish when ready
