Production Deployment

Multi-tenancy, failover, monitoring, high availability. Go live safely.

Pre-Production Checklist

  • ☐ All email services registered in DI
  • ☐ Credentials encrypted with production KeyVersion
  • ☐ Database indexes created (IX_AIExt_Credentials_Name_TenantID)
  • ☐ Load testing completed (5k+ emails)
  • ☐ Error logging configured
  • ☐ Monitoring/alerts set up
  • ☐ Rollback plan documented

HA Configuration

Multi-Provider Failover

-- Tenant has two credentials: primary + backup
INSERT INTO AIExt_Credentials (Name, ...) 
VALUES ('email-primary', ...);   -- Gmail

INSERT INTO AIExt_Credentials (Name, ...)
VALUES ('email-backup', ...);    -- SMTP

// In code:
var result = await _emailService.SendAsync(..., "email-primary", ...);
if (!result.Success)
{
    result = await _emailService.SendAsync(..., "email-backup", ...);
}

Database Performance

Index on credential lookups ensures <10ms response time:

CREATE NONCLUSTERED INDEX IX_AIExt_Credentials_Name_TenantID
ON AIExt_Credentials (Name ASC, TenantID ASC)
WHERE Deleted = 0
INCLUDE (CredentialID, DisplayMetadata, EncryptedData, ...)

Monitoring

Track these metrics:

  • Email send rate: emails/second
  • Success rate: % successful sends
  • Error rate: % failed sends
  • Response time: p50, p95, p99 latency
  • Queue depth: pending emails

Disaster Recovery

Provider Outage

Switch provider instantly:

-- Current credential fails
UPDATE AIExt_Credentials 
SET Enabled = 0 
WHERE Name='email-primary' AND TenantID=1;

-- Activate backup
UPDATE AIExt_Credentials 
SET Enabled = 1 
WHERE Name='email-backup' AND TenantID=1;

-- Restart app or wait for next request
// Application automatically uses enabled credential
