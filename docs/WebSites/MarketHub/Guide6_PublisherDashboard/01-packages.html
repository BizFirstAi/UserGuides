My Packages

The My Packages section gives publishers a complete view of every package in their portfolio — across all statuses — with actions for submitting, editing, versioning, and withdrawing packages.

Package Status Lifecycle

StatusDescriptionVisible to Public?Available Actions
DraftPackage is being prepared — not yet submitted for reviewNoEdit, Submit for Review, Delete
SubmittedAwaiting admin review; automated security scan has runNoView, Withdraw Submission
Under ReviewAssigned to an admin reviewer — review in progressNoView (read-only)

RejectedAdmin rejected — feedback providedNoView Feedback, Edit, Resubmit
DelistedRemoved from public marketplaceNoRelist (resubmit for review), Archive

Publishing a New Version

To publish a new version of an existing package:

1

Export from InstallHub

Use InstallHub to export the updated artifacts with the new version number. Download the updated package file.

2

Upload Package File

In My Packages, click "Publish New Version" for the relevant package. Upload the new package file from InstallHub.

3

Write Changelog

Add changelog text for the new version. Describe what changed, what was fixed, and any breaking changes.

4

Submit or Save as Draft

Submit for review (enters admin queue) or save as Draft for later. The currently published version remains live while the new version is under review.

Package Edit vs New Version
