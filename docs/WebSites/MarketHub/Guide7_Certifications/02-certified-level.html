Certified Level

Certified is the highest certification available to community publishers — earned through a structured human review process that validates documentation quality, test coverage, security posture, and issue management standards.

What Certified Signals

A Certified badge tells users that an admin reviewer has personally validated the package against a specific criteria scorecard — not just an automated scan. Users evaluating Certified packages can trust that:

  • The package is well-documented with an install guide and configuration reference
  • The publisher maintains meaningful test coverage (80%+ on node library components)
  • There are no known critical or high security vulnerabilities in the package's dependencies
  • The publisher has no open critical issues older than 30 days — they respond to bugs
  • The changelog is complete from initial release to current version

Review Timeline

Certified applications are reviewed within 3–5 business days of submission. The package remains live with Community certification during the review period — there is no status change visible to the public while the application is pending.
