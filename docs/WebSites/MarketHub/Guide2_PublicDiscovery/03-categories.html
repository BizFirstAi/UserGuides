Category Browsing

Categories organize the marketplace into curated collections by package type and domain — making it easy to browse focused areas of the ecosystem without a specific search query in mind.

Primary Categories

MarketHub has four primary package-type categories that map directly to the four types of BizFirstGO artifacts that can be packaged:

CategoryPackage TypeExample Packages
WorkflowsFlow Studio processes and templatesExpense approval, employee onboarding, document review workflows
FormsAtlas Form definitionsHR intake forms, customer onboarding forms, compliance checklists
AppsApp Studio applicationsInventory management app, project tracker, CRM lite
Node LibrariesCustom execution node collectionsStripe payment nodes, Salesforce sync nodes, data transformation nodes

Domain Sub-Categories

Within primary categories, domain sub-categories provide more specific browsing. Domain sub-categories are admin-configurable — new domains can be added as the ecosystem grows.

IntegrationsCRM connector nodes, ERP sync nodes, payment gateway nodes

Category Page Layout

Each category page includes:

  • Category header — name, description, and package count for this category
  • Featured packages — up to 3 admin-curated featured packages within the category
  • Sub-category filter tabs — one-click filter by domain sub-category
  • Package grid — paginated package cards sorted by the default relevance algorithm
  • Filter sidebar — same filter controls as the main search page, scoped to this category

Category Management

Categories and sub-categories are managed by marketplace admins in the Admin Portal under Rules Management. Publishers cannot create new categories — they select from the available list when publishing a package.

Single primary category per package

Each package is assigned exactly one primary category and zero or more domain sub-categories (max 3). A workflow package cannot appear in the Forms category. Cross-category mixed packages are listed in the primary artifact type's category.
