Issue Management

The admin issue management section provides oversight of all issues reported against marketplace packages — policy violations, user complaints, platform disputes, and escalated publisher issues.

Issue Types

Issue TypeWho ReportsWhat It Covers
Policy ViolationUsers or automated systemPackage content violates marketplace rules — e.g., malicious code, deceptive description
Package BugUsersTechnical defect in the package — reported against a specific version
Publisher ComplaintUsersIssues with publisher behavior — non-responsive support, abandoned packages
Escalated IssuePublisher or userIssues unresolved at publisher level escalated to admin for intervention
Platform IssuePublishersIssues with the marketplace platform itself — install failures, certification bugs

Issue Queue

The admin issue queue shows all open issues sorted by severity (Critical → High → Medium → Low) then by age (oldest first). Admins can filter by type, status, publisher, and severity.

ColumnDescription
SeverityCritical / High / Medium / Low — set by admin triaging, not auto-detected
TypePolicy Violation / Bug / Complaint / Escalated / Platform
PackageAffected package and publisher
ReporterUser who reported the issue
AgeDays since issue was opened
StatusOpen / Assigned / Escalated / Resolved / Closed

Admin Issue Actions

ActionDescription
Assign to SelfTake ownership of the issue — prevents duplicate admin work
Contact PublisherSend a formal admin message to the publisher regarding the issue
Delist PackageIf issue is a policy violation — delist while resolution is pending
ResolveMark issue resolved with resolution notes
Close (No Action)Close without action — e.g., duplicate report, invalid complaint
